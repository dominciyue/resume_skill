# 行为面 / HR 面（STAR 故事库）

大厂普遍有 HR 面/价值观面（字节 culture fit、阿里政委、亚马逊 LP），问"最大的
失败/最有成就感的事/和同事冲突/最难的决策"。很多人临场编，前后矛盾被识破。
这里从经历库提炼**可复用的真实 STAR 行为故事**，反复用于各种行为题。

## 第一步：建「行为故事库」（一次性，反哺经历库）

从经历库的「最难的点 / 我的动作 / 踩坑」字段，提炼一组可复用故事，每个按
STAR 结构整理：**情境 Situation → 任务 Task → 行动 Action（突出本人）→ 结果
Result（真实，暂缺则定性）→ 复盘 Reflection（学到什么）**。

覆盖这些高频维度，每维 1-2 个真实故事即可：
- 失败/挫折、冲突/分歧、抗压/紧急、主动/超出职责、领导/推动他人、从 0 到 1。

把整理好的故事追加进经历库新增小节「## 行为故事库」（和现有"漏洞反哺"机制
同构，越用越全）。**所有故事必须真实**，防虚构红线照旧。

## 第二步：行为面追问

按高频行为题调用对应故事，并追问验证真实性与深度：
- "这个失败具体怎么回事？→ 你当时怎么判断的？→ 为什么这么做不那么做？→
  结果到底如何、有没有量化？→ 现在回头看你会怎么改？"
- 识别两类问题：① 把团队功劳说成个人（对照经历库角色）；② "教科书式完美故事"
  （无真实挫折感）→ 追问细节逼出真实。

## 第三步：映射常见题

给用户一张"常见行为题 → 该用你哪个故事"的对应表，让他不必每题现想：

| 常见题 | 建议调用的故事 |
|--------|----------------|
| 最大的失败/挫折 | （指向故事库里的失败类） |
| 与同事意见冲突 | （冲突类） |
| 压力最大的一次 | （抗压类） |
| 最有成就感的事 | （从0到1/结果最硬的） |

结束转入 `interview/feedback.md`：行为面也产出"哪个故事讲得空、缺细节/缺反思"
的反馈，并把口头补充的真实细节反哺经历库。
