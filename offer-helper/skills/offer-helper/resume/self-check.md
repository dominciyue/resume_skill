# 简历交付前自检清单

生成简历后、交付给用户前，**逐项核对并在「匹配度报告」里输出本清单的勾选结果**。
这是把隐性标准变成显性 gate，保证换模型/换次生成质量都稳定可复现。
**任一项不通过 → 回改后重新输出整张表**，不要带着未过项交付。

## 内容
- [ ] 每段第一条 bullet 命中本次 JD 的高权重原词（针对性，见 star-rules 规则 7）
- [ ] 每条 bullet 有强动词开头 + 量化或定性结果，无"参与/负责"裸描述
- [ ] 数字尽量成对给基线（从 X 到 Y），无单独成行的孤立绝对数
- [ ] 无空话（性格开朗/吃苦耐劳/熟悉 Office 等，见 resume-quality 必删清单）
- [ ] 每段经历 3-5 条 bullet，经历多时只留最匹配的 3 段

## 防虚构
- [ ] 每条内容都能溯源到经历库原文（见 anti-fabrication.md）
- [ ] 经历库标「暂缺」处用定性表述，未编造任何数字

## 排版
- [ ] 单栏、无表格/图片/进度条；模块用标准名；联系方式在正文非页眉
- [ ] 应届 ≤1 页；社招若两页，无半页尾巴、entry 未被断页
- [ ] 全文无残留 `{{...}}` 占位符，无内容为空的 section

## 缺口提示与可执行性（关键）
- [ ] JD 高权重要求若经历库无证据，已在报告显著位置列出缺口，并**按性价比排序**
- [ ] 每条补齐建议都过 ACT：有具体动作（非仅缺口名词）/ 有时间或难度估计 / 有
      落地去向（写进简历哪段，带空位模板不替用户编）
- [ ] 报告以「下一步行动清单」收尾，每条带时间盒 + 具体动作 + 落地去向

## 输出格式
在匹配度报告末尾附一段，再接「下一步行动清单」：
> **交付自检**：内容 ✅ / 防虚构 ✅ / 排版 ✅ / 缺口可执行 ✅（逐项打勾，不过的项
> 说明已如何回改）
